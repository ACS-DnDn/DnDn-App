prod-api-alb WAF WebACL 생성 및 연결
Overview
조치 배경 SecurityHub WAF.1 Finding (HIGH) 감지 — prod-api-alb에 WAF 미연결로 외부 웹 공격 노출 상태
작업 내용 prod-api-alb에 WAFv2 WebACL 신규 생성 및 연결 (Count → Block 모드, Terraform)
근거 문서 이벤트 보고서 DOC-2026-03-001 우선순위 High
대상 리소스 prod-api-alb, aws_wafv2_web_acl (신규 생성) AWS 계정 123456789012 (dndn-prod)
작업 예정일 결재 완료 후 즉시 — 무중단 작업 (다운타임 없음) 담당자 devops@dndn
1. 작업 내용
구분 항목 내용
목적 작업 목적 WAF 미연결로 인한 외부 웹 공격 노출 해소 — WAF.1 Finding 해결 및 컴플라이언스 복구
범위 신규 생성 WAFv2 WebACL 생성
룰셋 적용 AWSManagedRulesCommonRuleSet + IP Reputation
연결 대상 prod-api-alb
방법 실행 Terraform IaC 적용
운영 전략 Count 모드 선 운영 후 오탐 없을 시 Block 모드 전환
일정 예상 소요 약 1~2시간 (Count 모드 모니터링 30분 포함)
2. 작업 절차
단계 작업 내용 실행 담당
① 사전 확인 CloudWatch ALB 트래픽 및 헬스체크 정상 확인, 비정상 요청 여부 사전 점검 수동 devops
② WAF 생성 aws_wafv2_web_acl 생성 — 초기 Count 모드 적용, 오탐 여부 확인 Terraform 자동
③ ALB 연결 aws_wafv2_web_acl_association으로 prod-api-alb에 WAF WebACL 연결 Terraform 자동
④ Count 모니터링 30분간 WAF 샘플링 로그 확인 — 정상 트래픽 차단 여부 및 공격 탐지 패턴 확인 수동 devops
⑤ Block 전환 오탐 없음 확인 후 override_action: none (Block 모드)으로 전환 Terraform 자동
⑥ 사후 모니터링 24시간 CloudWatch WAF 지표 추이 확인 (BlockedRequests / AllowedRequests) 수동 devops
⑦ Finding 확인 SecurityHub WAF.1 Finding 상태 PASSED 전환 확인 (최대 24시간 소요) 수동 devops
3. 변경 Before / After
항목 Before (현재 상태) After (적용 후)
WAF 연결 미연결 — WAF 없음 WAF WebACL (prod-api-waf) 연결
웹 공격 차단 미적용 AWSManagedRulesCommonRuleSet + IP Reputation 적용
WAF 로그 없음 CloudWatch Metrics + 샘플링 활성화
월 비용 WAF 비용 없음 +약 $25/월 (WebACL 기본 + 요청 과금)
다운타임 — 없음
SecurityHub 상태 WAF.1 FAILED WAF.1 PASSED
4. 위험도 분석
위험 항목 수준 분석
서비스 중단 하 WAF WebACL 연결은 무중단 — ALB 재시작 불필요, 트래픽 영향 없음
오탐 차단 (False Positive) 중 관리형 룰셋 초기 적용 시 정상 트래픽 차단 가능 — Count 모드 선 적용 후 Block 전환 권장
비용 증가 하 월 ~$25 추가 — 현재 트래픽 기준 산정, 예산 범위 내
롤백 가능성 하 Count 모드 전환 또는 WebACL 연결 해제로 즉시 원복 가능
5. 롤백 계획
롤백 트리거 WAF 적용 후 정상 트래픽 차단율 5% 초과 또는 서비스 응답 이상 감지 시
롤백 방법 1단계: WAF 룰 Count 모드 전환 (즉시, Terraform override_action { count {} } 재적용)
2단계: 오탐 원인 분석 후 룰 제외 처리 또는 WebACL 연결 해제
예상 시간 Count 모드 전환: 약 2~3분  /  WebACL 연결 해제: 약 1분
